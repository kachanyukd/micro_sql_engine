# 1. Загальний опис системи

Micro-SQL Engine — це консольна утиліта, яка дозволяє виконувати спрощені SQL-подібні запити до CSV-файлів.

Програма приймає:

* файл із даними (`CSV`)
* файл із запитом (`SQL`)

Система будує внутрішню модель запиту та виконує операції:

* вибір колонок
* фільтрацію рядків
* сортування результатів

---

# 2. Діаграма компонентів (Component Diagram)

Система поділена на декілька модулів.

## Основні компоненти

### CLI

Точка входу в програму.

Відповідає за:

* запуск програми
* отримання аргументів командного рядка
* передачу запиту у систему
* вивід результатів

---

### Query Parser

Модуль розбирає SQL-подібний запит.

Виділяє частини запиту:

* SELECT
* FROM
* WHERE
* ORDER BY

Результатом є внутрішня модель запиту.

---

### CSV Loader

Відповідає за зчитування CSV-файлів та створення таблиці даних у пам'яті.

---

### Query Model

Містить структуру запиту:

* список колонок
* джерело даних
* умови WHERE
* параметри сортування

---

### Execution Engine

Виконує запит над даними.

Функції:

* фільтрація рядків
* вибір колонок
* сортування результатів

---

### Result Formatter

Формує текстовий результат та передає його у CLI.

---

## Потік даних

```
Користувач
   ↓
CLI
   ↓
Query Parser
   ↓
Query Model
   ↓
CSV Loader
   ↓
Execution Engine
   ↓
Result Formatter
   ↓
CLI
   ↓
Результат
```

# 3. Діаграма класів (Class Diagram)

Основні класи системи:

### Query

Описує структуру SQL-запиту.

Містить:

* вибрані колонки
* джерело даних
* умови фільтрації
* параметри сортування

---

### Table

Представляє таблицю даних, завантажену з CSV.

---

### QueryParser

Розбирає текст SQL-запиту та створює об'єкт Query.

---

### Expression

Абстрактний клас для логічних виразів у WHERE.

Підтримує оператори:

* AND
* OR
* порівняння значень

---

### QueryExecutor

Виконує запит над таблицею.

---

### CSVLoader

Завантажує CSV-файл у таблицю.

---

### ResultFormatter

Формує результат виконання запиту для виводу.

# 4. Використані структури даних

## List

Використовується для:

* зберігання рядків таблиці
* формування результатів
* сортування даних

---

## Dictionary (Hash Map)

Використовується для представлення рядків таблиці.

Приклад:

```
{
  "id": 1,
  "name": "John",
  "salary": 5000
}
```

Доступ до даних виконується за ключем колонки.

---

## Expression Tree

Використовується для представлення умов WHERE.

Приклад:

```
       AND
      /   \
 salary>2000 role='user'
```

Це дозволяє обробляти складні логічні умови.
